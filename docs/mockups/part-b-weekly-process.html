The big picture

Workstream B3

Anonymous auto-tie — one deterministic path

One rule replaces the several assignment paths. It runs at submission and is re-checkable in Week Review. The anonymous submission page never shows prior picks; the tie only affects what the logged-in user and the leaderboard see.

Auto-tie logic
StepCheckResult
1Match email to users.email / users.leaguesafe_email / leaguesafe_payments.leaguesafe_emailFound → set assigned_user_id
2Is that user Paid for the season?Yes → show_on_leaderboard = true
3Unpaid or no matchshow_on_leaderboard = false → falls to Week Review queue
Hard requirement: the anonymous picks page loads a blank sheet every time, even for a returning email — it must never query or render previously submitted picks. When that person logs into their real account, their submitted picks appear there (Thu/Fri already locked, only Saturday editable).
Workstream B4

Disqualification — the 7-pick case

Submit Thursday → those are your 6. If a Thu/Fri pick locked and you then submit a full new Saturday set, you can end up with 7 counted picks. The system flags it after games complete and proposes the drop; the admin confirms in Week Review.
